# Requirements: Shop-Back (v1)

## v1 Requirements

### 1. Authentication & User (AUTH)
- [ ] **AUTH-01**: User can signup with email/password.
- [ ] **AUTH-02**: User can login and receive a JWT token.
- [ ] **AUTH-03**: User can fetch current account information.
- [ ] **USER-01**: User can view their order history.
- [ ] **USER-02**: User can update their profile information.

### 2. Product & Category (SHOP)
- [ ] **SHOP-01**: User can browse categories.
- [ ] **SHOP-02**: User can search products with filters (price, category, keyword).
- [ ] **SHOP-03**: User can view product details (description, price, stock status).
- [ ] **SHOP-04**: User can view product reviews and Q&A (read-only for v1).

### 3. Cart & Order (CART)
- [ ] **CART-01**: User can add/remove products to/from a persistent cart.
- [ ] **CART-02**: User can update cart item quantities.
- [ ] **ORDER-01**: User can place an order from their cart (checkout).
- [ ] **ORDER-02**: User can view details of a specific order.

### 4. Personalization (REC)
- [ ] **REC-01**: User can see personalized product recommendations based on simple heuristics (e.g., category interest).

## v2 Requirements (Deferred)
- [ ] OAuth integration (Google, GitHub).
- [ ] Real-time PG (Payment Gateway) integration.
- [ ] Multi-language and multi-currency support.
- [ ] Advanced recommendation engine (collaborative filtering).

## Out of Scope
- [ ] Admin CMS / Back-office portal.
- [ ] Real-time inventory tracking with external ERP.
