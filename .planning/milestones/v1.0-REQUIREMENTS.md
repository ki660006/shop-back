# Requirements: Shop-Back (v1.0)

## v1 Requirements

### 1. Authentication & User (AUTH)
- [x] **AUTH-01**: User can signup with email/password. — Implemented in Phase 1 (Wave 2): `POST /api/auth/signup`
- [x] **AUTH-02**: User can login and receive a JWT token. — Implemented in Phase 1 (Wave 2/3): `POST /api/auth/login` + `POST /api/auth/refresh` with DB-backed rotation
- [x] **AUTH-03**: User can fetch current account information. — Implemented in Phase 1 (Wave 3): `GET /api/users/me`
- [x] **USER-01**: User can view their order history. — Implemented in Phase 4 (Wave 3): `GET /api/orders`
- [x] **USER-02**: User can update their profile information. — Implemented in Phase 1 (Wave 3): `PUT /api/users/profile`

### 2. Product & Category (SHOP)
- [x] **SHOP-01**: User can browse categories. — Implemented in Phase 2 (Wave 1): `GET /api/categories` (hierarchical tree)
- [x] **SHOP-02**: User can search products with filters (price, category, keyword). — Implemented in Phase 2 (Wave 2): `GET /api/products` with PostgreSQL FTS + cursor pagination
- [x] **SHOP-03**: User can view product details (description, price, stock status). — Implemented in Phase 2 (Wave 2/3): `GET /api/products/{id}` with images
- [x] **SHOP-04**: User can view product reviews and Q&A (read-only for v1). — Implemented in Phase 2 (Wave 2): mock data in `ProductDetailResponse`

### 3. Cart & Order (CART)
- [x] **CART-01**: User can add/remove products to/from a persistent cart. — Implemented in Phase 3 (Wave 2): `POST/DELETE /api/cart/items` with guest support
- [x] **CART-02**: User can update cart item quantities. — Implemented in Phase 3 (Wave 2): `PATCH /api/cart/items/{id}`
- [x] **ORDER-01**: User can place an order from their cart (checkout). — Implemented in Phase 4 (Wave 2): `POST /api/orders` with pessimistic locking
- [x] **ORDER-02**: User can view details of a specific order. — Implemented in Phase 4 (Wave 3): `GET /api/orders/{id}`

### 4. Personalization (REC)
- [x] **REC-01**: User can see personalized product recommendations based on simple heuristics. — Implemented in Phase 5 (Wave 2): `GET /api/recommendations` (category-heuristic → top-selling → global fallback)

## v2 Requirements (Deferred — unchanged)
