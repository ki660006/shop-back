# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-05
**Phases:** 1–5
**Total Plans:** 15
**Code:** ~4,425 Java LOC

## Overview

Full e-commerce backend API for Shop-Front. Covers identity management, product catalog with search, persistent shopping cart, transactional checkout, and personalized recommendations.

## Phases

### Phase 1: Foundation & Authentication

**Goal**: Establish a secure backend foundation with identity management.
**Depends on**: Nothing
**Requirements**: AUTH-01, AUTH-02, AUTH-03, USER-02
**Plans**: 3 plans

- [x] 01-01: Project scaffolding, database setup (Flyway), and global error handling
- [x] 01-02: Spring Security 6.x configuration, JWT provider, signup/login endpoints
- [x] 01-03: User profile management and database-backed refresh token rotation

**Details:**
Spring Boot 3.4 + Java 21 Virtual Threads. JWT access tokens (30s) + DB-backed refresh rotation. Flyway V1 schema (users, refresh_tokens). Centralized `@RestControllerAdvice`. Full `AuthIntegrationTest` coverage.

---

### Phase 2: Product Catalog & Discovery

**Goal**: Implement a performant and searchable product catalog.
**Depends on**: Phase 1
**Requirements**: SHOP-01, SHOP-02, SHOP-03, SHOP-04
**Plans**: 3 plans

- [x] 02-01: Hierarchical category tree (recursive CTE) implementation
- [x] 02-02: Searchable product catalog with PostgreSQL FTS and cursor-based pagination
- [x] 02-03: Product image upload (Apache Tika MIME validation), file storage, and integration tests

**Details:**
UUID v7 (epoch-ordered) primary keys. PostgreSQL FTS via `search_vector @@ plainto_tsquery`. Multi-column cursor pagination (LATEST, PRICE_ASC/DESC, NAME_ASC) with Base64 encoding. Querydsl for type-safe dynamic queries. Mock reviews/Q&A data for SHOP-04. `CatalogIntegrationTest` coverage.

---

### Phase 3: Shopping Cart Persistence

**Goal**: Enable persistent cart state across browser sessions.
**Depends on**: Phase 1, Phase 2
**Requirements**: CART-01, CART-02
**Plans**: 3 plans

- [x] 03-01: Cart schema (Flyway V3), Cart/CartItem entities, ShedLock infrastructure
- [x] 03-02: Core CartService & API — CRUD, stock validation, 50-item/99-qty limits
- [x] 03-03: Cart merge logic (guest → member), daily cleanup batch, `CartIntegrationTest`

**Details:**
Guest carts identified by UUID cookie. Merge combines quantities (capped at 99), respects 50 unique item limit, deletes guest cart. ShedLock via `JdbcTemplateLockProvider` for distributed cleanup safety. `CartIntegrationTest` covers add, merge, update, delete, and stock validation.

---

### Phase 4: Checkout & Order Management

**Goal**: Complete the transactional loop for purchases and history.
**Depends on**: Phase 3
**Requirements**: ORDER-01, ORDER-02, USER-01
**Plans**: 3 plans

- [x] 04-01: Order/OrderItem/Payment schema (Flyway V4), entities, OrderNumberGenerator
- [x] 04-02: Checkout service — pessimistic lock stock deduction, mock payment, cart clear
- [x] 04-03: Order history, order details, cancellation with stock restoration, `OrderIntegrationTest`

**Details:**
`OrderNumberGenerator` produces `yyyyMMdd-ShortUUID` human-readable IDs. Pessimistic write lock on Product during checkout prevents overselling. `OrderIntegrationTest` includes concurrent checkout test proving lock correctness. Cancellation restores stock with `increaseStock()`.

---

### Phase 5: Personalization & Polish

**Goal**: Enhance user experience with recommendations and performance refinements.
**Depends on**: Phase 4
**Requirements**: REC-01
**Plans**: 3 plans

- [x] 05-01: Springdoc OpenAPI/Swagger setup, `@EnableCaching`, `recent_views` schema (Flyway V5)
- [x] 05-02: Hybrid recommendation engine (category-heuristic → top-selling → global fallback), daily view cleanup
- [x] 05-03: N+1 elimination (EntityGraph + JPQL JOIN FETCH), `RecommendationIntegrationTest`

**Details:**
`@Cacheable("categoryTree")` on category tree. `ConcurrentMapCacheManager` for in-process cache. `RecentViewCleanupBatch` trims to 50 most recent views per user via `ROW_NUMBER()`. N+1 fixed in `OrderRepository` (`findByUserId` + `@EntityGraph`, `findWithItemsById` + JPQL). `testCategoryCache()` verifies 0 DB hits on second fetch via Hibernate Statistics.

## Milestone Summary

**Key Decisions:**

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| Java 21 + Spring Boot 3.4 | Modern LTS with Virtual Thread support | ✓ Good |
| Modular Monolith (domain-grouped) | MVP speed + future scalability path | ✓ Good |
| PostgreSQL FTS + cursor pagination | Avoids full table scans, stable scroll | ✓ Good |
| UUID v7 (epoch-ordered) primary keys | Time-sortable, index-friendly | ✓ Good |
| Pessimistic locking on checkout | Prevents overselling in concurrent requests | ✓ Good |
| DB-backed refresh token rotation | Enables revocation, session tracking | ✓ Good |
| Mock payment (no PG) | v1 scope; real PG deferred to v2 | ✓ Good |
| ShedLock for batch cleanup | Distributed-safe cron without Redis | ✓ Good |
